# UI спецификация

## Навигация

- Три пункта: **Главная** (`/`), **Контакты** (`/contacts`), **Настройки** (`/settings`).
- Переключение: нижняя навигация на узких экранах; на `lg` и шире — фиксированная левая колонка (`src/app/layout/app-shell.tsx`, `AppNavLink`).
- Маршрут `/contacts/:contactId` (карточка контакта) и `/new` (создание долга) не имеют собственного пункта меню — активная вкладка определяется через `useActiveNavSection` (`src/app/layout/use-active-nav-section.ts`):
  - **Главная** активна на `/`, на `/new`, и на `/contacts/:id`, если на карточку перешли с главной;
  - **Контакты** активна на `/contacts`, и на `/contacts/:id`, если на карточку перешли со страницы «Контакты».
  - Источник перехода передаётся через `location.state.from` (`"home" | "contacts"`) в `<Link state={{ from }}>`; без состояния (прямой переход по URL) считается «Контакты» — по префиксу пути. См. `ContactListItem` (`entities/debt`), который принимает проп `linkState` и прокидывает его в `state`.

## Главная (`/`)

Виджет `widgets/home-screen` (**HomeScreen**): шапка с заголовком и кнопкой-ссылкой «Добавить долг» (→ `/new`), **HomeTotalsCards** (две карточки — «Мне должны» / «Я должен»: крупная сумма в валюте отображения + разбивка по валютам под ней), список контактов **ContactList** — **только контакты с непогашенным долгом** (мне или я им; закрытые долги не учитываются). Полный список контактов — на странице «Контакты».

## Контакты (`/contacts`, `widgets/contacts-screen`)

**ContactsScreen**: шапка с заголовком, список **всех** несархивированных контактов (`ContactListItem` с `linkState="contacts"`), включая тех, у кого нет непогашенных долгов. Пустое состояние — сообщение об отсутствии контактов.

## Создание долга (`/new`, `pages/create-debt` + `features/create-debt`)

Отдельная страница (не модалка) с кнопкой «назад»: **CreateDebtForm** — **ContactCombobox** (поиск существующих контактов / создание нового по введённому имени), переключатель направления (две кнопки «Мне должны» / «Я должен»), сумма (**AmountInput** — маска с разделением разрядов пробелом, `components/ui/amount-input`) + валюта (`Select` из RUB/USD/EUR/CNY), заметка (`Textarea`, опционально). Валидация: контакт обязателен, сумма > 0. При успехе — долг создаётся с операцией `initial` в истории, переход назад (на главную или на карточку контакта, если открыто оттуда через `?contactId=`).

Открывается: с главной (без параметра) или с карточки контакта (`/new?contactId=<id>`, кнопка «Добавить долг» там же). Кнопка «назад» и отмена ведут туда же, откуда открыли.

## Изменение долга (EditDebtDialog, `features/edit-debt`)

Модалка: те же поля, что при создании (направление, сумма, валюта, заметка), без выбора контакта. Направление/сумму/валюту можно менять только пока по долгу нет ни одного погашения (иначе поля задизейблены, показывается пояснение) — заметка редактируется всегда.

## Погашение (AddRepaymentDialog, `features/add-repayment`)

Модалка с текущим остатком долга, полем суммы (**AmountInput**, валидация: > 0 и ≤ остатку) и опциональной заметкой. При остатке 0 после операции — долг переходит в `closed`.

## Карточка контакта (`/contacts/:contactId`, `widgets/contact-detail`)

**ContactDetailScreen**: шапка с кнопкой «назад» (на главную или на «Контакты» — см. навигацию), аватаром и именем контакта, кнопкой-ссылкой «Добавить долг» (→ `/new?contactId=<id>`); две карточки сводки («Должен вам» / «Вы должны», сумма по валютам); список долгов контакта (новые сверху), каждый — **DebtCard**:

- направление + исходная сумма и валюта, заметка долга;
- кнопки «Изменить» (открывает **EditDebtDialog**) и «Удалить» (с подтверждением через `ConfirmActionModal`) — доступны **всегда**, включая закрытые (погашенные) долги;
- пометка «Погашен» рядом с кнопками для закрытых долгов;
- для активных — `Progress` (доля погашенного), остаток из скольки, кнопка «Погасить»;
- история операций — свёрнута по умолчанию (`Collapsible`, заголовок «Показать историю (N)»), внутри: тип операции, дата, сумма и заметка операции (если есть) — так отображаются в т.ч. заметки к погашениям, не загромождая карточку по умолчанию.

Несуществующий/удалённый `contactId` → **ContactNotFound** (сообщение + кнопка «на главную»).

## Экран настроек (`/settings`)

Виджет `widgets/settings-screen`:

| Блок | Описание |
|------|----------|
| Тема | Системная (по умолчанию), светлая, тёмная; хранение в `localStorage`, класс `dark` на `html`. |
| Валюта отображения | `Select` из RUB/USD/EUR/CNY (`features/set-display-currency`); влияет только на конвертацию итогов, не на хранимые долги. |
| Курс валют | Дата последнего обновления курса + кнопка ручного обновления (`features/refresh-exchange-rates`); при ошибке загрузки — предупреждение, что используется последний известный курс. |
| Язык | Селект задизейблен: зафиксирована русская локаль. |
| Резервная копия | Кнопки «Экспортировать JSON» / «Импортировать JSON»; импорт — через диалог подтверждения со сводкой (число контактов/долгов, дата экспорта) перед полной заменой данных. |
| Данные | Очистка IndexedDB, сброс `localStorage`, «очистить всё» с перезагрузкой страницы. Подтверждение через `AlertDialog`. |
| О приложении | Версия (`VITE_APP_VERSION`/`package.json`) и ссылка на репозиторий (`VITE_REPO_URL`/`package.json`, пусто, пока нет remote). |

## Гидрация и первый кадр

Пока данные из IndexedDB не загружены, контент заменён скелетонами; после `hydrated === true` — `fade-in`.

## Уведомления (toasts)

Глобальный **Toaster** (Sonner) подключён в `app/providers`; используется для результата экспорта/импорта резервной копии.

## i18n

Все пользовательские строки — через колоцированные `translations.ts` в слайсах + `shared/i18n` (`common`) для общих меток (сохранить/отмена/удалить/создать).
