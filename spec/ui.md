# UI спецификация

## Навигация (MVP)

- Два пункта: **Главная** (`/`), **Настройки** (`/settings`).
- Переключение: нижняя навигация на узких экранах; на `lg` и шире — фиксированная левая колонка (`src/app/layout/app-shell.tsx`).
- Карточка контакта — отдельный маршрут `/contacts/:contactId`, доступен только через клик по контакту на главной (в навигации не участвует).

## Главная (`/`)

Виджет `widgets/home-screen` (**HomeScreen**): шапка с заголовком и кнопкой «Добавить долг» (открывает **CreateDebtDialog**), **HomeTotalsCards** (две карточки — «Мне должны» / «Я должен»: крупная сумма в валюте отображения + разбивка по валютам под ней), список контактов **ContactList** (карточки-ссылки на `/contacts/:id`, нетто-остаток по направлениям под именем, аватар-инициал).

## Карточка долга — создание (CreateDebtDialog, `features/create-debt`)

Модалка (shadcn `Dialog`, вызов через `NiceModal.show`): **ContactCombobox** (поиск существующих контактов / создание нового по введённому имени), переключатель направления (две кнопки «Мне должны» / «Я должен»), сумма (`Input inputMode="decimal"`) + валюта (`Select` из RUB/USD/EUR/CNY), заметка (`Textarea`, опционально). Валидация: контакт обязателен, сумма > 0. При успехе — долг создаётся с операцией `initial` в истории.

## Погашение (AddRepaymentDialog, `features/add-repayment`)

Модалка с текущим остатком долга, полем суммы (валидация: > 0 и ≤ остатку) и опциональной заметкой. При остатке 0 после операции — долг переходит в `closed`.

## Карточка контакта (`/contacts/:contactId`, `widgets/contact-detail`)

**ContactDetailScreen**: шапка с кнопкой «назад» на главную, аватаром и именем контакта, кнопкой «Добавить долг» (предзаполненный контакт); две карточки сводки («Должен вам» / «Вы должны», сумма по валютам); список долгов контакта (новые сверху) — каждая карточка: направление + исходная сумма и валюта, заметка, для активных — `Progress` (доля погашенного), остаток из скольки, кнопка «Погасить», кнопка удаления долга (с подтверждением через `ConfirmActionModal`); для закрытых — пометка «Погашен» вместо кнопки удаления; под каждым долгом — лента истории операций (тип, дата, сумма).

Несуществующий/удалённый `contactId` → **ContactNotFound** (сообщение + кнопка «на главную»).

## Экран настроек (`/settings`)

Виджет `widgets/settings-screen`:

| Блок | Описание |
|------|----------|
| Тема | Системная (по умолчанию), светлая, тёмная; хранение в `localStorage`, класс `dark` на `html`. |
| Валюта отображения | `Select` из RUB/USD/EUR/CNY (`features/set-display-currency`); влияет только на конвертацию итогов, не на хранимые долги. |
| Курс валют | Дата последнего обновления курса + кнопка ручного обновления (`features/refresh-exchange-rates`); при ошибке загрузки — предупреждение, что используется последний известный курс. |
| Язык | Селект задизейблен: зафиксирована русская локаль. |
| Резервная копия | Кнопки «Экспортировать JSON» / «Импортировать JSON»; импорт — через диалог подтверждения со сводкой (число контактов/долгов, дата экспорта) перед полной заменой данных. |
| Данные | Очистка IndexedDB, сброс `localStorage`, «очистить всё» с перезагрузкой страницы. Подтверждение через `AlertDialog`. |
| О приложении | Версия (`VITE_APP_VERSION`/`package.json`) и ссылка на репозиторий (`VITE_REPO_URL`/`package.json`, пусто, пока нет remote). |

## Гидрация и первый кадр

Пока данные из IndexedDB не загружены, контент заменён скелетонами; после `hydrated === true` — `fade-in`.

## Уведомления (toasts)

Глобальный **Toaster** (Sonner) подключён в `app/providers`; используется для результата экспорта/импорта резервной копии.

## i18n

Все пользовательские строки — через колоцированные `translations.ts` в слайсах + `shared/i18n` (`common`) для общих меток (сохранить/отмена/удалить/создать).
