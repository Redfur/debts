# Функциональные требования

## User stories

1. Как пользователь, я хочу видеть на главном экране, сколько всего мне должны и сколько должен я, чтобы быстро оценить свою общую позицию по долгам.
2. Как пользователь, я хочу создать новый долг на человека, указав направление (я должен / мне должны), сумму и валюту, без лишних полей.
3. Как пользователь, я хочу при создании долга выбрать уже существующий контакт из списка, а не вводить имя заново каждый раз.
4. Как пользователь, я хочу видеть по каждому контакту суммарный долг и открыть карточку с историей операций по всем его долгам.
5. Как пользователь, я хочу частично или полностью погасить долг и видеть остаток после каждого погашения.
6. Как пользователь, я хочу задавать валюту для каждой записи долга индивидуально и видеть агрегированный итог в одной выбранной валюте отображения.
7. Как пользователь, я хочу пользоваться приложением без сети (PWA), а курс валют — подтягивать из интернета не чаще раза в день с офлайн-фолбэком.

## Функциональные требования (текущий продукт)

- **Контакты**: создаются неявно при создании долга (по имени, `ContactCombobox` — поиск среди существующих + создание нового); отдельный экран со списком всех контактов — `/contacts` (без создания/редактирования контакта напрямую).
- **Долг**: `contactId`, направление (`i_owe` | `owed_to_me`), валюта (RUB/USD/EUR/CNY), исходная сумма, заметка (опционально), статус (`active` | `closed`). Создаётся на отдельной странице `/new` (не в модалке); ввод суммы — с маской разделения разрядов (`AmountInput`).
- **Изменение долга**: направление, сумма, валюта и заметка редактируются через `EditDebtDialog`; направление/сумму/валюту можно менять только пока по долгу нет ни одного погашения (после первого погашения — только заметка).
- **Погашение**: частичное или полное; сумма не может превышать текущий остаток; при остатке 0 долг переходит в `closed`.
- **История операций**: у каждого долга — лента операций (`initial` при создании, `repayment` при каждом погашении) с суммой, датой и заметкой операции (если указана); отображается в свёрнутом по умолчанию блоке на карточке долга.
- **Итоги на главной**: остатки активных долгов группируются по валюте и направлению, конвертируются в валюту отображения (настройка в `/settings`, по умолчанию RUB) по курсу из `ratesCache`. На главной показываются только контакты с непогашенным долгом; полный список — на `/contacts`.
- **Курс валют**: обновляется не чаще раза в сутки автоматически при старте приложения; ручное обновление — в настройках; последний известный курс кэшируется в IndexedDB и используется офлайн.
- **Удаление долга**: доступно для любого долга, включая погашенные (`closed`); удаляет долг и всю его историю операций.
- **Резервная копия данных**: в `/settings` — экспорт/импорт JSON (контакты, долги, операции, `meta`) с подтверждением перед полной заменой данных.

## Non-goals (сейчас не делаем)

- Сервер, аккаунты, синхронизация между устройствами.
- Создание/редактирование/архивирование контактов отдельно от долгов (только через сценарий создания долга).
- Операция «корректировка» долга (кроме `initial`/`repayment`).
- Напоминания и сроки погашения (due date), push-уведомления.
- Пользовательский выбор набора поддерживаемых валют (пока фиксированный список RUB/USD/EUR/CNY).

## Edge cases

| Ситуация | Ожидание |
|----------|----------|
| **Погашение больше остатка** | Форма отклоняет ввод с сообщением об ошибке; операция не создаётся. |
| **Нет курса для валюты** | Итог на главной по этой валюте не конвертируется (`totalOwedToMeConverted`/`totalOwedByMeConverted` = `null`, показывается «—»), но сумма видна в разбивке по валютам. |
| **Офлайн при старте** | `refreshRatesIfStale` падает тихо (`ratesFetchError`), используется последний закэшированный `ratesCache`; при полном отсутствии кэша — конвертация недоступна, работает разбивка по валютам. |
| **Повреждённое/частично прочитанное хранилище** | Ошибка чтения: сообщение в store (`lastError`), возможность сброса БД из настроек. |
| **Пустые данные** | Нет контактов/долгов — дружелюбный empty state на главной с призывом добавить первый долг. |
| **Импорт резервной копии с неверным форматом** | Toast с ошибкой парсинга/валидации; текущие данные не затрагиваются. |
