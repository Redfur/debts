# Целевое решение — обзор

## Назначение

Локальное PWA-приложение для учёта долгов между пользователем и другими людьми («кто кому должен»), без бэкенда: кто должен пользователю, кому должен пользователь, история операций по каждому долгу.

## Поток: открытие до создания долга

1. Пользователь открывает приложение (онлайн не обязателен).
2. `app` монтируется, провайдеры (i18n, тема, NiceModal) активны.
3. Store (`useDebtStore`) вызывает `hydrate()`: `StorageAdapter` читает контакты, долги, операции и `meta` (в т.ч. кэш курса валют) из IndexedDB; после гидратации в фоне запускается `refreshRatesIfStale()`.
4. **Главная** (`/`): после гидратации — **HomeTotalsCards** (конвертированные итоги «мне должны» / «я должен» в валюте отображения + разбивка по валютам) и **ContactList** (контакты с нетто-остатком).
5. Кнопка «Добавить долг» открывает модалку **CreateDebtDialog**: выбор/создание контакта (**ContactCombobox**), направление, сумма, валюта, заметка.
6. Погашение — модалка **AddRepaymentDialog** на карточке долга в **ContactDetailScreen** (`/contacts/:id`): сумма не больше остатка; при полном погашении долг переходит в статус `closed`.

## Основные сценарии

| Сценарий | Шаги |
|----------|------|
| Новый долг | Кнопка «Добавить долг» → выбор контакта (или создание нового по имени) → направление, сумма, валюта → `createDebt` в store → операция `initial` в истории. |
| Погашение | Карточка контакта → долг → «Погасить» → сумма (валидация ≤ остатку) → `addRepayment`; при остатке 0 — статус `closed`. |
| Просмотр контакта | Главная → карточка контакта → сводка по направлениям + список долгов с прогрессом и историей операций. |
| Итоги на главной | Остатки активных долгов группируются по валюте и направлению, конвертируются в валюту отображения по кэшированному курсу (`ratesCache`); при отсутствии курса для валюты — «—» вместо суммы, но разбивка по валютам видна всегда. |
| Обновление курса | Не чаще раза в сутки автоматически при гидратации; вручную — кнопка в настройках. Офлайн — используется последний закэшированный курс. |
| Резервная копия данных | `/settings` → «Экспортировать JSON» / «Импортировать JSON» — локально, без сети. |

## Границы системы

- Данные (контакты, долги, операции) остаются на устройстве; сеть используется только для курса валют (опционально, с офлайн-фолбэком).
- Один пользователь на профиль браузера (без аккаунта).
