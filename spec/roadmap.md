# Расширяемость и этапы развития

## 1. Текущая база (MVP)

- Главная: конвертированные итоги «мне должны»/«я должен» + список контактов с нетто-остатком.
- Создание долга: выбор/создание контакта, направление, сумма, валюта (RUB/USD/EUR/CNY), заметка.
- Погашение (частичное/полное) с валидацией остатка; авто-закрытие долга при остатке 0.
- Карточка контакта: сводка по направлениям, список долгов с прогрессом и историей операций, удаление долга.
- Курс валют: авто-обновление раз в сутки + ручное обновление в настройках, офлайн-фолбэк на кэш.
- Настройки: тема, валюта отображения, статус курса, резервная копия (JSON export/import), сброс данных.
- PWA: офлайн после первой загрузки, `site.webmanifest`, Service Worker precache.

## 2. Контакты

- Отдельный экран/редактирование контакта (переименование, ручной выбор цвета аватара).
- Архивирование контактов без долгов.

## 3. Долги — расширение модели

- Операция «корректировка» (изменение исходной суммы без создания нового долга).
- Сроки погашения (due date) и напоминания.
- Прикрепление заметок/вложений к операциям.

## 4. Валюты

- Пользовательский выбор набора поддерживаемых валют (сейчас фиксированный список RUB/USD/EUR/CNY).
- Отображение истории курса, а не только последнего значения.

## 5. Backend + sync (опционально, позже)

- `ApiStorageAdapter` с тем же интерфейсом `StorageAdapter`.
- Аккаунт, синхронизация между устройствами.

## 6. CI/CD

- `.github/workflows/ci.yml` — `check`/`knip`/`build` на каждый push.
- `.github/workflows/release.yml` + `scripts/release.sh` — тег → GitHub Release → деплой на GitHub Pages. Требует git remote на GitHub; подключить, когда проект будет готов к публикации.
